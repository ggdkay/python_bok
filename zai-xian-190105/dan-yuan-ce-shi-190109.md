* 语句覆盖

##### 语句覆盖，代码覆盖到哪行，就会覆盖行数比上所有的语句数量

```
漏洞： and  -> or ,行覆盖发现不了问题，多种方法测试同一个功能，构造不同的输入
if a=2 and b>3 
if a=2 or b>3
```

##### 条件覆盖，覆盖所有的条件

```
条件1 a=2，T，F
条件2 b>3，T，F

漏洞
测试用例指数集增长= 2*条件
测试用例的有效性，排列组合耗费资源，尽量少的case覆盖多的代码
```

##### 判断覆盖，某个判断语句击中

```
a == 2 or  x>1 判断语句已经覆盖
a ==2 ,x>1 两个条件都需要，才是条件覆盖

漏洞
a == 2 or x>1 => a==2 or x<1

条件覆盖是判断覆盖的字集
满足其中一个条件即可满足判断覆盖了
```

##### 路径覆盖

```
路径覆盖在判断，条件都会覆盖到
每一个可能的路径都会覆盖
```
